Visão Geral

O módulo IxcService do BR10ACS fornece integração avançada com a API REST do IXC Soft, cobrindo consulta de usuários RADIUS, ONTs fibra autorizadas, dados de sinal óptico e importação automática de coleções de API. Todos os endpoints requerem autenticação JWT via header Authorization: Bearer <token>.

🔐
Autenticação

O sistema usa duas camadas de autenticação distintas.

1. JWT — BR10ACS (todas as requisições à API interna)

Obtenha o token via POST /v1/auth/login e inclua em todas as requisições:

Header obrigatório
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

2. Basic Auth — IXC Soft (gerenciado internamente pelo BR10ACS)

O token IXC é armazenado criptografado na integração e usado automaticamente. O formato do token é userId:hash — obtido nas configurações de API do IXC Soft.

Formato do token IXC
// Exemplo de token IXC (userId:hash)
6:4dacdb8e47193e8cbbabe508c3c59b4547e463817b1d9b9a1d20ab4812fe1a62

// O BR10ACS converte para Base64 e envia como:
Authorization: Basic NjozZGFjZGI4ZTQ3MTkzZThjYmJhYmU1MDhj...
⚠️
O token IXC nunca é exposto na interface ou nas respostas da API. Ele é armazenado criptografado no MongoDB e usado apenas internamente pelo IxcService.

Erros e Logs

Todos os erros de comunicação com o IXC são registrados no MongoDB e visíveis na tela de Logs do BR10ACS.

Formato de resposta de erro

{
  "found": false,
  "error": "Integração não encontrada"
}
Situação Nível de Log Categoria Mensagem
Integração não encontrada — — Retorna found: false sem log
Integração desabilitada — — Retorna found: false sem log
Erro HTTP na API IXC ERROR INTEGRATION Mensagem do erro + incrementa stats.errors
Teste de conexão OK INFO INTEGRATION Latência em ms + status HTTP
Teste de conexão falhou WARN INTEGRATION Mensagem do erro de conexão

Buscar Usuário RADIUS

Consulta a tabela radusuarios do IXC Soft. Retorna dados de login PPPoE, status de conexão, IP atual, MAC, concentrador, VLAN e informações da ONT associada.

GET /v1/integrations/:id/ixc/rad-user Buscar usuário RADIUS por login, MAC ou MAC da ONT ▾
Parâmetros
Exemplo
Resposta
ParâmetroTipoObrigatórioDescrição
:id path Sim ObjectId MongoDB da integração IXC configurada no BR10ACS
login query Opcional* Login PPPoE do usuário (ex: cliente001)
mac query Opcional* MAC do equipamento (ex: AA:BB:CC:DD:EE:FF)
onuMac query Opcional* MAC da ONT (campo onu_mac no IXC)
⚠️
* Pelo menos um dos parâmetros de busca (login, mac ou onuMac) deve ser informado. A prioridade é: login > mac > onuMac.
Busca por login PPPoE
GET /v1/integrations/6847a1b2c3d4e5f6a7b8c9d0/ixc/rad-user?login=cliente001
Authorization: Bearer eyJhbGciOiJIUzI1NiJ9...
Busca por MAC da ONT
GET /v1/integrations/6847a1b2c3d4e5f6a7b8c9d0/ixc/rad-user?onuMac=48:57:02:AA:BB:CC
Authorization: Bearer eyJhbGciOiJIUzI1NiJ9...
Exemplo com fetch (JavaScript)
// Busca usuário RADIUS por login PPPoE
const response = await fetch(
  '/api/v1/integrations/6847a1b2c3d4e5f6a7b8c9d0/ixc/rad-user?login=cliente001',
  {
    headers: {
      'Authorization': `Bearer ${token}`
    }
  }
);
const data = await response.json();
if (data.found) {
  console.log('Login:', data.data.login);
  console.log('IP atual:', data.data.ip);
  console.log('Online:', data.data.online === 'SS');
}

GET /v1/integrations/:id/ixc/ont-complete Busca completa: radusuarios + radpop_radio_cliente_fibra ▾
Parâmetros
Exemplo
Resposta

:id path Sim ObjectId da integração IXC
login query Opcional* Login PPPoE do usuário
mac query Opcional* MAC do equipamento ou da ONT

Authorization: Bearer eyJhbGciOiJIUzI1NiJ9...

Atualizar Sinal da ONT

Atualiza os dados de sinal óptico de uma ONT fibra diretamente no IXC Soft. Requer perfil Admin ou Super Admin.

PUT /v1/integrations/:id/ixc/ont-signal/:ontId Atualizar sinal_rx, sinal_tx, temperatura e voltagem de uma ONT ▾
Parâmetros
Exemplo
Resposta
ParâmetroTipoObrigatórioDescrição
:id path Sim ObjectId da integração IXC
:ontId path Sim ID da ONT na tabela radpop_radio_cliente_fibra do IXC
sinal_rx body Opcional Sinal recebido em dBm (ex: "-22.50")
sinal_tx body Opcional Sinal transmitido em dBm (ex: "2.30")
temperatura body Opcional Temperatura em °C (ex: "42.5")
voltagem body Opcional Voltagem em V (ex: "3.28")
data_sinal body Opcional Data/hora da medição (formato: YYYY-MM-DD HH:mm:ss). Se omitido, usa a data/hora atual.
Atualizar sinal da ONT ID 387
PUT /v1/integrations/6847a1b2c3d4e5f6a7b8c9d0/ixc/ont-signal/387
Authorization: Bearer eyJhbGciOiJIUzI1NiJ9...
Content-Type: application/json

{
  "sinal_rx": "-22.50",
  "sinal_tx": "2.30",
  "temperatura": "42.5",
  "voltagem": "3.28"
}
Resposta — Sucesso (200 OK)
Resposta — Erro
{
  "ok": false,
  "error": "Request failed with status code 404"
}
